# What's new in ALP-aca v1.2?

Release candidate 2, 22 April 2026.

## ALP-aca logo in plots

Now you can add the cool<sup>TM</sup> ALP-aca logo to the cool<sup>TM</sup> plots you generate with ALP-aca!

In `exclusionplot` and `alp_channels_plot`, the argument `logo_position` controls the position of the logo. It can be:

* `0` or `'best'` (only `matplotlib`): best position (default)
* `1` or `'upper right'`: upper right
* `2` or `'upper left'`: upper left
* `3` or `'lower left'`: lower left
* `4` or `'lower right'`: lower right

For example:

```python
exclusionplot(..., logo_position=1)
```

However, if your plot is uncool<sup>TM</sup>, you can remove the logo by setting `logo_position=None`.

## Total $\chi^2$

The objects `ChiSquared` and `ChiSquaredList` now have a `.chi2_tot()` method, which returns the total $\chi^2$ of the object.

Additionally, the computation of the number of sigmas with `.significance()` method has been improved for points with large likelihoods.

## Measurements included in each sector

The `Sector` class now implements the method `.get_measurements()`, that returns a dict whose keys are a (observable, experiment) tuple, and values are the `MeasurementBase` object.

## Callbacks

In functions that calculate multiple parameter points, now it is possible to include a callback function that is executed after the calculation of each single point. The callback function _does not_ modify the result, its main purpose is informative (for example, to implement progress bars). The signature must be `callback(**info)`, where the dict `info` will contain the following keys: `process`, `result`, `ma`, `couplings`, `fa`, `br_dark`, and the optional arguments used for RGE evolution (if any). The functions that implement callbacks are

* `alpaca.decay_width`
* `alpaca.branching_ratio`
* `alpaca.cross_section`
* `alpaca.meson_mixing`
* `alpaca.alp_channels_decay_widths`
* `alpaca.alp_channels_branching_ratios`
* `alpaca.statistics.get_chi2`
* and the corresponding `Scan` methods.

Calling these functions without the `callback` argument will revert to the behaviour of previous versions.

Example of a simple callback function that reports the last point calculated:

```python
fa = np.logspace(3, 8, 100)

def callback(**info):
    print(f'fa={info["fa"]:.4e} GeV: Result = {info["result"]:.4e} GeV')

dws = alpaca.decay_width('alp -> pi0 pi+ pi-', ma=2.0, couplings=coupling, fa=fa, callback=callback)
```

Example of a callback that implements a counter

```python
class CallbackCounter:
    def __init__(self):
        self.count = 0

    def __call__(self, **info):
        self.count += 1
        print(f'Callback called {self.count} times. Current fa={info["fa"]:.4e} GeV. Decay width: {info["result"]:.4e} GeV.')

dws = alpaca.decay_width('alp -> pi0 pi+ pi-', ma=2.0, couplings=coupling, fa=fa, callback=CallbackCounter())
```

Example of a callback implementing a simple progressbar using `tqdm` (requires additional library)

```python
from tqdm import tqdm

with tqdm(total=len(fa)) as pbar:
    callback_tqdm = lambda **kwargs: pbar.update() # Fix callback signature
    dws = alpaca.decay_width('alp -> pi0 pi+ pi-', ma=2.0, couplings=coupling, fa=fa, callback=callback_tqdm)
```

Example of a callback implementing a simple progressbar using `rich` (requires additional library)

```python
from rich.progress import Progress

with Progress() as progress:
    task = progress.add_task("Calculating decay widths...", total=len(fa))
    callback_rich = lambda **kwargs: progress.update(task, advance=1) # Fix callback signature
    dws = alpaca.decay_width('alp -> pi0 pi+ pi-', ma=2.0, couplings=coupling, fa=fa, callback=callback_rich)
```

Example of a callback implementing a progressbar with additional information using `rich` (requires additional library)

```python
from rich.live import Live
from rich.table import Table
from rich.console import Group
from rich.progress import Progress, BarColumn, TaskProgressColumn, TimeRemainingColumn

progress = Progress(
    TextColumn("[progress.description]{task.description}"),
    BarColumn(),
    TaskProgressColumn(),
    TimeRemainingColumn(),
)

def make_display(progress, info):
    table = Table.grid(padding=(0,2))
    table.add_column(justify="left", style='bold')
    table.add_column(justify='right')
    table.add_row("Current fa:", f"{info['fa']:.4e} GeV")
    table.add_row("Decay width:", f"{info['result']:.4e} GeV")
    return Group(progress, table)

class CallbackLive:
    def __init__(self, progress, live):
        self.progress = progress
        self.live = live

    def __call__(self, **kwargs):
        self.progress.advance(0)
        self.live.update(make_display(self.progress, kwargs))

info = {'fa': np.nan, 'result': np.nan} # Initialize with NaN to avoid issues with uninitialized values
with Live(make_display(progress, info), refresh_per_second=10) as live:
    task = progress.add_task("Calculating a -> 3pi...", total=len(fa))

    pbar_rich = CallbackLive(progress, live)
    dws = alpaca.decay_width('alp -> pi0 pi+ pi-', ma=2.0, couplings=coupling, fa=fa, callback=pbar_rich)
```

## RNG seed for `vegas`

The algorithm `vegas`, used for numerical integration of the phase space in some of the hadronic decays, relies on RNG (Monte Carlo integration), and consequently, some results in previous versions might vary between different executions of ALP-aca. As of this version, ALP-aca now fixes the seed of `vegas` to guarantee deterministic results. The default seed is 12345. You can change it with

```python
alpaca.config.vegas_seed = 42
```

And if you want to change it persistently,

```python
alpaca.config.save()
```

In ALP-aca, the results of several computations are stored in cache, so if you call again with the same parameters, the observable is not recalculated. If you want to force the recalculation (_warning_: of all observables!) in order to check the effect of changing the seed, try

```python
alpaca.clear_cache_dw()
```

## Bug and compatibility issues fixed

* A bug that ignored optional parameters in several functions from being passed to `ALPcouplings.match_run` has been fixed. In particular, using `integrator = 'no-rge'` now correctly computes the matching at the high scale without running the couplings down to the weak scale, with a notable speedup in the computation of the exclusion limits. (08e63d9c1a90d9a24b609176ab202c19cbda9acf)
* Various bugs affecting ALPs heavier than the EW scale have been fixed. (b2a704c79ff92af8690630814e204471f5b39017)
* `ALPcouplings` now accept `numpy` types. (00d1b78f84050594b138a4f37d59607b3b48bc8a)
* Compatibilty issues with newer versions of `scipy` (>=1.16.0) have been fixed. (1c310c2a5740c65ff1562fedcf902e75bb471dc6)
* Broken `flavio` dependencies have been fixed. (595b9fe7ba77d392b4068b4bfb97b24dfced8e10)
* A bug appearing when using an `int` as the final scale for RGE running. (1df208d6564d3bd01a551e8a5972d20ffa7f6b38)
* Optional dependencies not installed correctly. (729fa6c635986383e2d1efc69bdb8aa18de892d5)
* A bug in the interpolation of some upper limits for displaced vertex decays (e.g. $B^+ \to K^+ \gamma \gamma$). (3a670009a8bc6777c3f1b3a56e13d74c6183b45c, 1d9b0f26ac0acf5309e850a72608607cd357b04d)
